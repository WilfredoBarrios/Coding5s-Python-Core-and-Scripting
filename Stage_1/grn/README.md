## 📚 Coding5s Methodology (5 Stages)

Ñande approach ha'e **[Controlled Cognitive Friction] (explained as [ñemoarandu sasõ oñembohape]) ryepýpe oñembohape**. Sapa lab nde po rupi ogueraha:

| Stage | Name | Objective |
|-------|------|-----------|
| **1** | Practice | [Code] (explained as [tembiaporã ñembohape]) oñepyrũ guive ehai |
| **2** | Debug | Intentional [syntax] (explained as [ñe'ẽ ñembohape]) [errors] (explained as [pantã]) logic ryepýpe ehesa'ỹijo |
| **3** | Complete | Architectural algorithmic gaps oñemohenda |
| **4** | Refactor | Existing [scripts] (explained as [tembiaporã ñembohape]) optimize ruwa (flat vs. modular structure) |
| **5** | Extend | Musuq production-ready functionalities oñembojoaju |

*Kóva repository ha'e **Stage 1 (Practice)** ryepýpe oĩ*.

## 🛠️ Mba'éichapa sapa Livebook oñemomba'eguasu?

Sapa `.livemd` archivo ryepýpe, kóva simple workflow nde po rupi ogueraha:

1. Archivo [Livebook] (explained as [kuaa tembiapo rupa]) ryepýpe ebojuaju (desktop térã browser version).
2. **Stage 1 prompt** ecopy ruwa, upéi nde preferred AI chat (ChatGPT, Gemini, Claude, etc.) ryepýpe epegue.
3. AI **3 guive 5 peve executable micro‑examples** topic ryepýpe ogenera.
4. Kóva examples Livebook **execution cell** ryepýpe ecopy ruwa, upéi erun.
5. [Code] (explained as [tembiaporã ñembohape]) emodifica, experimenta, upéi output real time ryepýpe oñembohasa ehesa'ỹijo.

Sapa Livebook **[Mentor prompt] (explained as [yanapã tapuy ñembohape])** ryepýpe oĩ. Kóva same AI chat ryepýpe nde [code] (explained as [tembiaporã ñembohape]) logic nde py'aguapy explanation ndive epegue. Mentor:

- **Ndaha'éi kutichĩ ome'ẽva directamente** (ñepyrũ guive)
- Socratic questions tapu pantã nde kúi descubrihaguã
- Real‑world analogies **[programming concepts] (explained as [tembiaporã ñe'ẽ ypykue])** oñembo'e haguã oipuru
- "Vibe Coding" chinkachĩ, upéi true engineering intuition oñembohape haguã
- Senior‑level answers specific questions tapu ryepýpe ome'ẽ

| **5** | The Strategic Partner | Visionary, professional "system architect" | Edge cases, fault tolerance, business logic alignment |
