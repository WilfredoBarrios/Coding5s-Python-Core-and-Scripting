## 🚀 Wie starte ich?

Um dieses Lab interaktiv zu erleben, müssen Sie **Livebook** auf Ihrem Computer installiert haben, zusammen mit der Python-Runtime-Integration.

1. **Livebook installieren:** Laden Sie es unter [livebook.dev](https://livebook.dev) herunter.
2. **Ein Lab öffnen:** Durchsuchen Sie die folgenden Ordner und klicken Sie auf die Schaltfläche **"Run in Livebook"** in jeder Datei, oder importieren Sie die URL direkt in Ihre Livebook-Sitzung.

---

## 📚 Coding5s-Methodik (5 Phasen)

Unser Ansatz basiert auf **gesteuerter kognitiver Reibung** (Controlled Cognitive Friction). Jedes Lab führt Sie durch die folgenden Schritte:

| Phase | Name | Ziel |
|-------|------|------|
| **1** | Praxis | Code von Grund auf neu schreiben |
| **2** | Debuggen | Absichtlich eingebaute Syntax- und Logikfehler finden |
| **3** | Vervollständigen | Architektonische und algorithmische Lücken füllen |
| **4** | Refactoring | Bestehende Skripte optimieren (flache vs. modulare Struktur) |
| **5** | Erweitern | Neue, produktionsbereite Funktionalitäten hinzufügen |

*Dieses Repository enthält **Phase 1 (Praxis)**.*

## 🛠️ Wie wird jedes Livebook verwendet?

Folgen Sie innerhalb jeder `.livemd`-Datei diesem einfachen Arbeitsablauf:

1. Öffnen Sie die Datei in **Livebook** (Desktop- oder Browserversion).
2. Kopieren Sie den **Prompt für Phase 1** und fügen Sie ihn in Ihren bevorzugten AI-Chat (ChatGPT, Gemini, Claude usw.) ein.
3. Die KI generiert **3 bis 5 ausführbare Mikro-Beispiele** zu diesem Thema.
4. Kopieren Sie diese Beispiele in die **Ausführungszelle** (execution cell) von Livebook und führen Sie sie aus.
5. Ändern Sie den Code, experimentieren Sie und beobachten Sie, wie sich die Ausgabe in Echtzeit verändert.

---

## 🧠 Der Sokratische Mentor (KI)

Jedes Livebook enthält außerdem einen **Mentor-Prompt**. Sie können ihn zusammen mit Ihrem Code und einer kurzen Erklärung Ihrer Logik in denselben AI-Chat einfügen. Der Mentor wird:

- Ihnen die Antwort **nicht direkt geben** (anfangs)
- Ihnen sokratische Fragen stellen, damit Sie den Fehler selbst entdecken
- Analogien aus der realen Welt nutzen, um **Programmierkonzepte** zu erklären
- Ihnen helfen, "Vibe Coding" (Codieren nach Gefühl) zu eliminieren und echte ingenieurtechnische Intuition aufzubauen
- Antworten auf Senior-Niveau liefern, wenn Sie spezifische Fragen stellen

## 🏛️ Architektur & Mentoren

Jedes Lab enthält ein **Erklärungs-Tor** (Explanation Gate), an dem ein spezialisierter KI-Mentor Sie durch den Lernprozess führt. Jede Phase hat ihre eigene Mentor-Persönlichkeit, die auf die psychologischen und pädagogischen Bedürfnisse dieser Phase abgestimmt ist:

| Phase | Mentor | Persönlichkeit | Was er lehrt |
|-------|--------|----------------|--------------|
| **1** | Der sichere Begleiter | Herzlich, geduldig, wie ein älteres Geschwisterteil | Sicheres Erkunden der Syntax, Eliminierung von "Vibe Coding" |
| **2** | Der analytische Partner | Neugierig, kaffeegetrieben, ein "Log-Detektiv" | Lesen von Fehlermeldungen (error traces), Verstehen von Systemabstürzen |
| **3** | Das logische Gerüst | Methodisch, ermutigend, ein "Senior-Erbauer" | Visualisierung des Datenflusses (Input → Transform → Output) |
| **4** | Der elegante Kritiker | Sarkastisch, puristisch, "harte Liebe" (tough love) | Idiomatischen Code, funktionale Muster, Eliminierung schlechter Angewohnheiten |
| **5** | Der strategische Partner | Visionär, professionell, ein "Systemarchitekt" | Randfälle (edge cases), Fehlertoleranz, Ausrichtung auf die Geschäftslogik |

*Dieses Repository enthält **Phase 1 (Der sichere Begleiter)**.*
